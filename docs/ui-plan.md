这个产品最优的界面形态不是一个单页大盘，也不是纯向导，而应是 **PC-first 的研究工作台 + 配置中心 + 结果校正中心** 三位一体。也就是说，用户不是只“发起一次搜索然后看报告”，而是在一个持续研究环境里，完成“定义问题—配置平台—并行采集—校正结果—形成判断—输出方案”的完整闭环。

## 总体结构

建议整个应用采用 **三层界面架构**，这样最符合你这个产品的复杂度，也最适合 Tauri 桌面应用的工作台形态。

## 第一层：应用级主壳

这是整个程序的最外层框架，用来组织全局导航、项目切换、搜索状态和系统设置。复杂 B2B/SaaS 工作台一般采用稳定侧边栏和顶部全局状态区，以支持多模块切换和高密度信息。

建议结构：

- 左侧：一级导航栏

- 顶部：全局搜索、当前项目、后台任务状态、设置入口

- 中部：当前页面主内容

- 右侧：上下文侧栏，显示证据、说明、操作建议或任务明细

## 第二层：项目级研究工作台

这是用户真正工作的地方。每个项目都是一个完整研究容器，包含：

- 想法定义

- 平台配置

- 搜索任务

- 竞品池

- 用户声音

- 评估结果

- 策略建议

- 验证计划

## 第三层：对象级详情层

任何重要对象都需要可下钻的详情视图，而不是停留在列表：

- 某个平台配置详情

- 某个竞品详情

- 某个主题聚类详情

- 某条原始证据详情

- 某个评分维度详情

建议左侧一级导航固定如下：

| 一级模块  | 作用                           |
| ----- | ---------------------------- |
| 首页    | 项目列表、最近研究、模板、快速创建            |

- **四层搜索词汇包**（替代原有的单一"搜索关键词包"）：

  - 痛苦表达词汇（新增）：用户感知痛苦时的原始搜索语言

- 采纳

- 排除

- 手动新增

- 标记为邻近类

每条词汇允许用户：

- 排除（不用于第二轮）
- 手动修改
- 标记为高价值

此区域的标题可显示为"从第一轮搜索中发现的社区原生词汇"，帮助用户理解这些词汇来自真实用户讨论，而非系统臆测。

## 全局右侧上下文栏

为了让复杂界面不失控，我建议全局保留一个可折叠右侧上下文栏。它根据当前页面动态显示：

- 当前页面说明

- 当前对象详情

- 原始证据预览

- 当前配置解释

- 推荐下一步动作

- 注意事项 / 风险提醒

这对研究型产品很有用，因为用户经常需要在“当前操作”和“上下文解释”之间来回切换。
